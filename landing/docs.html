Documentation

Everything you need to know about installing, setting up, navigating modes, running commands, analyzing outputs, managing sessions, and privacy.

Install & Launch

Install Git-Reverse globally using npm to access it from anywhere in your terminal.

$ npm install -g git-reverse-cli --force

To update the CLI to the latest version:

$ npm update -g git-reverse-cli

To launch the interactive dashboard:

$ git-reverse

First Launch

On your first run, the CLI will prompt you for your setup details. These are stored locally.

Note
You will need an OpenRouter API Key to access free and premium LLMs. A direct link is provided during setup.
  • Username: How the CLI addresses you.
  • OpenRouter API Key: Used to access free and premium LLMs.
  • Model Selection: Choose a default model for your queries.

Analyze

Start an analysis by providing a target repository or local folder.

# Analyze a GitHub repository
$ git-reverse https://github.com/vitejs/vite

# Analyze the current local directory
$ git-reverse .

# Analyze with an initial query
$ git-reverse . "Explain the authentication flow"

Modes

Toggle between modes seamlessly within the CLI using the Tab key.

  • Standard / Reverse: Generates prompts to rebuild the project.
  • Deep-dive: Detailed learning path and scaling pitfall analysis.
  • Compact: Highly dense bulleted architectural digest.
  • Explore: Ask LLMs to explain files, architecture, and design patterns.

Commands

Type a forward slash / or backslash \ in the interactive prompt to run commands.

  • /settings — Modify API keys and models.
  • /update — Force update the CLI globally.
  • /deepdive — Toggle deep-dive mode.
  • /compact — Toggle compact mode.
  • /sessions — List all saved runs.
  • /resume <id> — Resume a past session.
  • /quit — Save history and exit.

Output

The CLI generates a structured recreation prompt which includes:

  • Project Overview & Tech Stack
  • System Architecture
  • Key Files Content (bypassing ignores)
  • Design Decisions & Build Process
  • Step-by-step Recreation Instructions

Sessions

Every interaction is automatically saved locally with an 8-character ID.

Tip
You can resume any session right from the terminal using the --resume flag!
$ git-reverse --resume a1b2c3d4

Privacy

All configurations and sessions are stored strictly on your local machine. No telemetry or tracking is included.

Caution
Since your API keys are stored locally, be careful not to share your configuration files if you stream your terminal.
  • Linux/Mac: ~/.config/git-reverse/
  • Windows: %APPDATA%\git-reverse\
